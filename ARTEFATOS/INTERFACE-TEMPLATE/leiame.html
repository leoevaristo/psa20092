Orientações para montagem dos formulários

É importante que sigam à risca todas as orientaçõs deste documento, pois como o css do formulário está padronizado se houver alguma modificação o layout irá "quebrar".

  • Só existirá um Form;
  • Poderá existir vários "fieldsets", as tags "fieldset" servem para separar o conteúdo do form por grupo. Observando o modelo percebe-se que existe dois fieldsets, cada um com uma tag "legend" com suas descrições "grupo 01" e "grupo 02" respectivamente. Utilizando um exemplo real pode-se conciderar "Dados Pessoais" um grupo "Informações Gerais" para outro grupo;
  • A estrutura de uma linha do form está da seguinte forma: <span class="linhaForm"><label for="texto" class="lab">texto</label><input id="texto" name="texto" class="inp" type="text" /></span>
    • A tag "span" representa uma linha, nela contém um atributo "class" que não deve ser alterada;
    • Dentro da tag "span" existem as tags "label" e "input", no caso dos inputs, eles podem ser modificados pelos "select" e "textarea" como contém no modelo padão;
    • Na tag "label" também não se deve mexer no atributo "class", já no atribulo "for" será informado o nome do campo desejado, por exemplo for="nome" ou for="idade". Dentro da tag onde contém a palavra "texto" será substituido pelo nome do campo, esse é o texto que será mostrado para o usuário;
    • Na tag "input" também não se deve mexer no atributo "class", já no atributo "id" será informado o mesmo nome que foi dado ao atributo "for" do label, exemplo id="nome" para for="nome", essa ligação será percebida quando o usuário clicar no "texto" que existe dentro da tag "label". Uma vez clicado irá ativar um "setfocus" para o input em questão;
  • É importante observar que todas as linhas seguem uma mesma estrutura e que NUNCA se deve modificar os atributos "class";
  • Para as tags "input" que tiverem o atributo "type" com a propriedade "radio" é importante sempre deixar o atributo "name" igual para todos, fazendo assim que todos pertençam ao mesmo grupo.
